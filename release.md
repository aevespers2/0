# Release Plan

## Principles
A release may advance only from completed `taskchain.md` and `punchlist.md` work with reproducible evidence.

## States
`DRAFT` · `GATED` · `READY` · `RELEASED` · `BLOCKED` · `ROLLED BACK`

## Release Gates
- [ ] Scope and version are defined.
- [ ] Required tasks are DONE and linked to commits.
- [ ] Build, lint, type checks, and tests pass.
- [ ] Security and dependency checks pass or approved exceptions are documented.
- [ ] Documentation, examples, changelog, and migration notes are current.
- [ ] Artifacts are reproducible, checksummed, and provenance-linked.
- [ ] Compatibility and rollback criteria are documented.
- [ ] Release approval is recorded.
